Description: There is a web server on port :80 protected with Port Knocking. Find the one “knock” needed (sending a SYN to a single port, not a sequence) so you can curl localhost.

Port Knocking 是一种网络安全技术，用于隐蔽地开启防火墙上的端口。它的核心思想是：只有按特定顺序访问一系列封闭的端口后，目标服务器才会开放真正的服务端口。
       工作原理

  1. 所有端口默认关闭：服务器上的关键端口（如 SSH 的 22 端口）在防火墙上默认是关闭的，防止扫描和攻击。
           发送“敲门序列”：客户端必须按照预先设定的顺序，向某些特定端口发送连接尝试（如 TCP SYN、UDP 包等）。
           例如，先访问端口 1001，再访问 2002，最后访问 3003。
  2. 服务器检测敲门序列：服务器上的守护程序（如 knockd）会监听这些尝试，如果顺序正确，就临时开放目标端口（如 SSH 的 22 端口）。
  3. 连接目标服务：客户端现在可以正常连接目标端口（如 SSH）。
  4. 端口自动关闭：一段时间后或连接结束后，端口再次关闭，防止被扫描或攻击。
コマンド 用途
ping パケットを送受信し、通信相手と通信できるか確認する
curl URLを指定してファイルを送受信する
nmap ポートをスキャンする
ss 「ソケット」の情報を出力する
コマンド 用途
df ディスクの空き容量を一覧表示する
lsblk ブロックデバイスを一覧表示する
