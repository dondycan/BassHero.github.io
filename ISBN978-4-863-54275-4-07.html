MMU(Memory Management unit):メモリ管理ユニット
仮想アドレス空間から物理アドレス空間の変換を行う機能がある、物理メモリをページと呼ばれる最小単位で管理できる。ソフトウェアのプロセスごとにページ単位でメモリ割り当てする。
ページテーブルに参照する際に仮想空間に対応する物理空間が見つからないと「ページフォルト」が発生します。
ページアウトが発生すると不要とされるページを二次記憶に書き出して（ページアウト）、新たに必要なページを二次記憶装置から物理空間に読み込んで(ページイン)、ページを置換する。

パソコンやサーバは起動するとまず HDD にインストールされた OS (Kernel) プログラムをメモリ領域に展開します。このとき使われるメモリ領域を 『カーネル空間』と呼びます。
一方、OS 上でプログラムが起動された場合は OS の管理下の元、メモリ領域に展開します。このときのメモリ領域を 『ユーザー空間』j7と呼びます。

ユーザー空間:

  1. プロセスごとにメモリ割り当てする。
  2. 物理メモリを意識する必要がない。
                  ▶物理メモリの上限以上にメモリを利用可能
                  ▶物理空間で断片化したメモリを連続するメモリとして扱える。
  3. 0~3GBの空間にマッピング

カーネル空間:

  1. 物理メモリにストレートマップ
  2. 多重化されない
  3. ３GB以降の空間にマッピング

プロセス

プロセス ・プログラムの実行単位
・1つのプロセスは、1つのメモリ空間を持つ
              ▶異なるプロセス間では変数アクセス不可 ・プロセスは、task_struct構造体（task_t型）で管理
・プロセスには必ず親プロセスがいる
        ※プロセス作成の仕組み
                ①システムコールfork（）で子プロセスを作成
                ②exec（）で新しいプログラムを実行
                ③子プロセスはexit（）で終了
                ④親プロセスはwait*（）で子プロセスの終了ステータスを取得
              ▶親がwaitを呼ばないとゾンビになる（子が終了しない）
・子プロセスは親プロセスの複製なので、ファイルのオープン情報やセマフォなどの状況も引き継がれる
              ▶デッドロックなどプログラムが停止する原因に注意が必要 ・データ領域（ヒープ、スタック）は親と子で別々になっている

スレッド

●プログラムの実行はプロセスと同様
       ・スレッドの動作に必要な情報は単独で持つ （thread ID、スケジュール優先度、スケジュールポリシーなど）
       ・プロセスの情報（スレッド固有でないもの）は共有する （プロセスID、親プロセスID、プロセスグループID、ユーザーID、グループID、カレントディレクトリ位置など）
       ・スレッドもtask_struct構造体（task_t型）を持つ
●スレッド間共有資源を操作するときは排他的制御が必要
●POSIX互換のpthread*で作成、終了

カーネル

コマンドuname -r で動いているカーネルのバージョンを表示する

アーキテクチャ種類 内容
システムコールインタフェース カーネルが持つ機能をユーザ空間のプロセスから利用できるよう提供するインタフェース。
カーネル・コード ハードウェアのアーキテクチャに依存しない、LinuxOSがサポートするCPUアーキテクチャ全てに共通する部分。プロセス管理やスレッド管理のスケジューラーなど。
アーキテクチャ依存のコード 特定のアーキテクチャに依存した固有のコード部分。ARMやIntelといったCPU、SoCのアーキテクチャに依存するコードなど。

カーネル関連のソースコードディレクトリ

パス 説明
arch アーキテクチャ依存部x86やArm、Power、MIPSなどがディレクトリで用意されている
block ハードディスクドライブ/メモリ領域などのアドレス指定可能な機器
crypto 暗号/復号化レイヤ
drivers 各種ドライバLinuxカーネルの半分の規模を占める
fs ファイルシステムextやfat、nfsなどがディレクトリで用意されている
include ヘッダファイル群アーキテクチャ依存部はarch/xxx/includeに配置
init 初期化処理
ipc プロセス間通信レイヤ
kernel カーネルコア部スケジューラやタイマーなどが含まれる
lib 共通ライブラリ対象アーキテクチャで高速化命令が存在しない場合などに使用されることが多いe.g. memcpy、memcmp、strcmp
mm メモリ管理アーキテクチャ非依存部アーキテクチャ依存部はarch/xxx/mmに配置
net 各種ネットワークe.g. ipv4/6、bluetooth、wimax
security セキュリティSELinuxやtomoyoなど
sound ALSAのカーネル側コード

VFS(Virtual File System)

VFSとは標準的なUNIXファイルシステムのすべてのシステムコールを取り扱う、カーネルが提供するソフトウェアレイヤです。

デバイスファイル /dev

原則として、一つのデバイスに対して、一つのデバイスファイルが存在する。

システムファイル /sys

/sysディレクトリはLinuxカーネルの実体のない仮想ファイルシステムであり、デバイスやカーネルモジュールの情報を表示することや、カーネルやファイルシステムなどの設定値の変更などができます
